EOS Core

EOS Core e un kernel bare metal scritto interamente in E — zero C, zero assembly oltre il multiboot header. Boota direttamente su x86 via QEMU o hardware fisico con Multiboot.

Funzionalita del kernel

  • Driver PS/2 tastiera: polling con mappatura scancode → ASCII. Supporto per shift, caps lock, tasti estesi (frecce, Home, End, Delete).
  • Output VGA text mode: buffer 80×25 a 0xB8000. Cursore hardware via registri VGA (0x3D4/0x3D5). Scroll automatico.
  • Shell interattiva: parser comandi, history buffer con UP/DOWN, line editing con LEFT/RIGHT/Home/End/Delete, backspace.
  • REPL aritmetico: valutazione live di espressioni con +, -, *, /. Operatori bitwise. Letterali esadecimali.
  • Comandi built-in: help, clear, echo, halt, reboot, version, scancode (debug).

Architettura

Il kernel e compilato da un singolo file .eee in LLVM IR, poi tradotto in codice nativo i686 da llc. Il linker produce un eseguibile ELF32 con header Multiboot che QEMU carica direttamente.

Pipeline di build:

kernel.eee  →  [e build --kernel]  →  kernel.ll  →  [llc-18]  →  kernel.o  →  [ld]  →  kernel.elf

Il kernel alloca tutto su stack (alloca LLVM). Non c'e heap, non c'e malloc. I buffer sono array statici E (mem N).

Stack tecnologico

  • Linguaggio: E (compilatore Rust → LLVM IR)
  • Target: i686-unknown-elf (32-bit x86 bare metal)
  • Boot: Multiboot 1 via header nel file ELF
  • I/O: Port I/O x86 per tastiera PS/2 e VGA
  • Testing: QEMU con output VGA e monitor QMP
