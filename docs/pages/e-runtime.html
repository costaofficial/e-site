e-runtime

Pipeline di compilazione

Il compilatore trasforma il sorgente E in codice nativo attraverso questi passaggi:

  1. Lexer: tokenizzazione del sorgente .eee in token (keyword, numeri, stringhe, operatori).
  2. Parser: parsing ricorsivo discendente che produce un AST (Abstract Syntax Tree).
  3. Compiler: walk dell'AST e generazione di LLVM IR. Gestisce scope, tipi, espressioni, controllo di flusso.
  4. LLVM: llc compila l'IR in codice macchina per l'architettura target.

Modalita di esecuzione

  • User mode: compila per x86_64-pc-linux-gnu con libc. Supporta printf, malloc, strlen. Usato per script e automazione.
  • Kernel mode (--kernel): compila per i686-unknown-elf senza libc. Nessuna dipendenza runtime. Built-in per I/O bare metal (inb, outb, VGA, PS/2).
  • Dry-run: interpreta lo script senza compilare. Output di log per validazione. Comando e run.

Target supportati

  • x86_64-pc-linux-gnu — user mode, compilazione nativa
  • i686-unknown-elf — kernel mode, bare metal, zero dipendenze

Il compilatore e scritto in Rust e genera LLVM IR tramite emissione diretta di istruzioni LLVM. In kernel mode, funzioni come vga_clear, kb_read_raw, inb/outb sono tradotte in assembly inline LLVM.
