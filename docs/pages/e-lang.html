e-lang

e-lang e un linguaggio di programmazione per sistemi e automazione. Compila direttamente in LLVM IR e puo girare su bare metal. Sintassi leggibile con blocchi do ... done, dati strutturati, operatori bitwise, e array statici su stack.

Variabili e scope

Le variabili si dichiarano con let. Lo scope e lessicale: le variabili dichiarate dentro un blocco { } sono visibili solo al suo interno e shadowano quelle esterne.

let x = 5
let y = x + 10

when x > 3 {
  let x = 99       // shadowa x esterno
  log x            // stampa 99
}
log x              // stampa 5 (scope esterno)

Tipi di dato

E supporta data per tipi strutturati e mem per array statici su stack.

data (record)

data Point x y
data KeyEvent scancode ascii flags

let p = Point 10 20
let v = p.x              // accesso ai campi con il punto
let q = Point p.x (p.y + 5)

mem (array statici)

let buf = mem 256       // 256 elementi su stack, zero-inizializzati
let val = buf[0]         // lettura per indice
let history = mem 1024   // perfetto per buffer kernel

Operatori

Aritmetici e bitwise

let a = 10 + 3          // addizione
let b = 10 - 3          // sottrazione
let c = 10 * 3          // moltiplicazione
let d = 10 / 3          // divisione intera

let e = 0xFF & 0x0F     // bitwise AND  → 15
let f = 0xF0 | 0x0F     // bitwise OR   → 255
let g = 0xFF ^ 0x0F     // bitwise XOR  → 240
let h = ~15             // bitwise NOT  → -16
let i = 1 << 3          // shift left   → 8
let j = 16 >> 2         // shift right  → 4

Confronto e logici

when x == 5 and y != 10 do log "ok" done
when x > 3 or x < 0 do log "range" done
when not (x > 10) do log "not" done

Controllo di flusso

when / or (if / else)

when x > 5 {
  log "big"
} or when x > 2 {
  log "medium"
} or {
  log "small"
}

// Sintassi alternativa con do...done
when x > 5 do
  log "big"
done

while / stop / skip

let i = 0
while i < 10 do
  when i == 5 do stop done    // esce dal loop
  when i == 3 do skip done    // salta alla prossima iterazione
  let i = i + 1
done

Modalita kernel (bare metal)

Con il flag --kernel, E compila per i686-unknown-elf senza libc. Le seguenti primitive sono disponibili come built-in che generano LLVM IR inline:

  • kb_init _ — abilita controller PS/2 tastiera
  • kb_read_raw _ — poll del byte allo stato della porta 0x64, restituisce scancode
  • vga_clear _ — pulisce buffer VGA 80x25 a 0xB8000
  • vga_raw_write char offset color — scrive carattere e attributo in VGA
  • vga_string str x y color — scrive stringa in posizione VGA
  • vga_scroll _ — scrolla lo schermo di una riga verso l'alto
  • vga_read_char row col — legge carattere dalla memoria VGA
  • eval_simple row start end — valuta espressione aritmetica dalla riga VGA
  • print_num row col color value — stampa numero decimale su VGA
  • inb port / outb port val — I/O porte x86
  • reboot _ — reset CPU via controller tastiera (0xFE su porta 0x64)
  • halt _ — loop infinito (ferma il kernel)

Esempio: kernel minimo EOS

data Cursor x y

fn set_cursor cur do
  let pos = cur.y * 80 + cur.x
  let lo = pos - (pos / 256) * 256
  let hi = pos / 256
  outb 0x3D4 0x0F
  outb 0x3D5 lo
  outb 0x3D4 0x0E
  outb 0x3D5 hi
done

fn main _ do
  let cur = Cursor 2 3
  let history = mem 1024
  vga_clear _
  kb_init _

  while 1 do
    let sc = kb_read_raw _
    when sc == 0x1C do           // Enter
      let cur = Cursor 2 (cur.y + 1)
      set_cursor cur
    done
  done
done

Build

e build --kernel kernel.eee -o kernel.ll
llc-18 -filetype=obj kernel.ll -o kernel.o
ld -m elf_i386 -T linker.ld kernel.o -o kernel.elf
qemu-system-x86_64 -kernel kernel.elf
